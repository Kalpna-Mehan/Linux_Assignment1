Question 2 – Secure Project Workspace Setup

1. Directory Structure Created

Commands:
mkdir ProjectWorkspace
cd ProjectWorkspace
mkdir Documents SourceCode
touch Documents/plan.txt SourceCode/main.c
ls -R

Output:
.:
Documents  SourceCode

./Documents:
plan.txt

./SourceCode:
main.c

Observation:
A secure project workspace was created with separate folders for documents and source code. This structure helps organize project files efficiently.

--------------------------------------------------

2. File Permissions Before Modification

Commands:
umask
ls -ld .
ls -l
ls -l Documents
ls -l SourceCode

Output:
0022
drwxr-xr-x 4 kalpna kalpna 4096 Jul  3 05:13 .
total 8
drwxr-xr-x 2 kalpna kalpna 4096 Jul  3 05:13 Documents
drwxr-xr-x 2 kalpna kalpna 4096 Jul  3 05:13 SourceCode
total 0
-rw-r--r-- 1 kalpna kalpna 0 Jul  3 05:13 plan.txt
total 0
-rw-r--r-- 1 kalpna kalpna 0 Jul  3 05:13 main.c

Observation:
The initial permissions were assigned automatically according to the system umask value (0022). Directories default to 755 and files default to 644, which are the default Linux permissions after the umask is applied.

--------------------------------------------------

3. File Permissions After Modification

Commands:
chmod 700 Documents
chmod 600 Documents/plan.txt
chmod 755 SourceCode
chmod 644 SourceCode/main.c
ls -ld Documents
ls -l Documents
ls -ld SourceCode
ls -l SourceCode

Output:
drwx------ 2 kalpna kalpna 4096 Jul  3 05:13 Documents
total 0
-rw------- 1 kalpna kalpna 0 Jul  3 05:13 plan.txt
drwxr-xr-x 2 kalpna kalpna 4096 Jul  3 05:13 SourceCode
total 0
-rw-r--r-- 1 kalpna kalpna 0 Jul  3 05:13 main.c

Observation:
Permissions were modified to improve security. The Documents folder (700) is now accessible only by the owner, while SourceCode (755) remains readable and executable by others. The sensitive plan.txt file (600) is restricted to owner read/write only, while main.c (644) stays readable by group and others since source code is meant to be shared.

--------------------------------------------------

4. Ownership Details

Output:
Owner: kalpna
Group: kalpna
(confirmed via the "kalpna kalpna" fields in the ls -l outputs above)

Observation:
The owner and group for all created files and directories are both "kalpna," since the workspace was created directly by that user account with no explicit chown needed.

--------------------------------------------------

5. Umask Value

Command:
umask

Output:
0022

Observation:
The umask value 0022 removes write permission for the group and others when new files and directories are created. This provides secure default permissions (755 for directories, 644 for files) without needing manual chmod for every new item.

--------------------------------------------------

6. How Permissions Protect Project Data

Linux file permissions control who can read, write, and execute files and directories. Restricting Documents to owner-only access (700/600) prevents unauthorized users from viewing or modifying sensitive project files, while keeping SourceCode more open (755/644) allows collaborators to read and use the code as intended. This selective restriction protects confidential data while preserving necessary collaboration.

--------------------------------------------------

Conclusion

The secure project workspace was successfully created. Directory permissions, ownership, and umask settings were verified and configured to protect project files.
